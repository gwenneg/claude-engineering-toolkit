---
name: generate-coderabbit-config
description: Generate a .coderabbit.yaml config that points CodeRabbit to existing agent guideline docs
---

Generate a `.coderabbit.yaml` configuration file that configures CodeRabbit to use the domain-specific guideline files produced by `/generate-agent-guidelines`. $ARGUMENTS

Whenever you need to ask the user a question, always use the AskUserQuestion tool — never ask as plain text.

## 1. Check for guideline files

Use Glob to check if any `docs/*-guidelines.md` files exist in the repository.

If no guideline files are found:

- Inform the user that guideline docs are required for this skill to work
- Suggest running `/generate-agent-guidelines` first to create them
- Stop here — do not generate any configuration

If guideline files are found, list them and proceed to the next step.

## 2. Generate the CodeRabbit configuration

Build a minimal `.coderabbit.yaml` with only the fields that differ from CodeRabbit defaults. The generated file must contain:

1. A schema comment on the first line:
   ```yaml
   # yaml-language-server: $schema=https://coderabbit.ai/integrations/schema.v2.json
   ```

2. A reference comment on the second line:
   ```yaml
   # See https://docs.coderabbit.ai/reference/configuration for all fields and default values
   ```

3. The `knowledge_base` section pointing to the guideline files:
   ```yaml
   knowledge_base:
     code_guidelines:
       filePatterns:
         - "docs/*-guidelines.md"
   ```

Do not include any other fields or sections — only what differs from CodeRabbit defaults.

## 3. Handle existing configuration

If a `.coderabbit.yaml` already exists at the repo root:

1. Read the existing file
2. Merge the new `knowledge_base.code_guidelines.filePatterns` entry into the existing configuration
3. Preserve all existing settings — do not remove or overwrite them
4. If the existing file already contains a `knowledge_base.code_guidelines.filePatterns` entry, merge the patterns (avoid duplicates)
5. Ensure the schema comment and reference comment are present at the top of the file, adding them if missing

If no `.coderabbit.yaml` exists, create a new one with the content from step 2.

## 4. Create a pull request (optional)

After the `.coderabbit.yaml` has been written, ask the user if they want to create a pull request with the changes. If they decline, stop here.

If they want a PR:

1. Create a new branch named `add-coderabbit-config`
2. Stage the `.coderabbit.yaml` file
3. Commit with a descriptive message summarizing what was configured
4. Push the branch and create a pull request using `gh pr create`
5. The PR description must include: `This PR was generated by the /generate-coderabbit-config Claude skill from https://github.com/gwenneg/claude-engineering-toolkit`
6. Display the PR link in the chat
